## 推荐阅读路径

- **新加入的开发**: 架构设计 → 项目能力与演进方向 → 后端代码导读 → Raft原理详解
- **运维人员**: 架构设计(第一节看目标、第五节看技术栈) → 部署指南
- **想深入分布式**: 架构设计(第四节) → Raft原理详解 → 后端代码导读(store/fsm 部分)
- **想扩展平台支持**: 架构设计 → platform-abstraction-design → 后端代码导读(platform 部分)

## 当前阶段

**v0.6.7 已完成**。单节点控制面、3 节点容错集群、Agent 骨架、登录鉴权、三路配置比对(含语义级值差异)、Agent 自动扫描与生效配置合并(部署后事件触发)、扩容迁移编排、G6 拓扑可视化、SSH 自动注入部署、入口代理、YAML 配置文件启动、服务器管理重构、跨平台命令执行抽象层、Go embed 单二进制、平台 Builder 单元测试、资源监控与告警(CPU/内存/磁盘/网络/负载采集 + 内存环形缓冲 + ECharts 曲线 + 阈值告警 Webhook)、操作审计日志(独立 bbolt 存储 + 写操作自动记录 + 敏感操作标记 + 操作日志页)、配置中心化与版本管理(配置基准走 Raft + 版本历史回滚 + 一键下发到 Agent 本地文件)、Agent 热更新与版本管理(GET_VERSION 上报 + UPGRADE 在线下载替换 + 滚动批量升级 + 版本兼容性提示)、更多中间件管理(进程特征识别 Redis/PostgreSQL/MySQL/Kafka/Zookeeper/ES/ClickHouse + MiddlewareOps 扩展点 + 前端类型标签分色)全部就绪。
