# 部署指南

本文档说明 deepsea-ops 在 Linux 服务器集群上的打包与部署。对应架构见 [架构设计.md](./架构设计.md)。当前处于 v0.4 阶段(3 节点容错集群 + SSH 自动注入 + 入口代理), 本文档覆盖单节点起步、3 节点集群扩展、SSH 自动注入部署三种模式。

---

## 一、部署架构概览

```
                ┌──────────────────────────────┐
   浏览器 ─────▶│  控制面节点 (3 台, v0.2 起)   │
                │  deepsea-server               │
                │  HTTP:8080  gRPC:9090  Raft:7000 │
                │  + 前端静态文件(nginx 托管)   │
                └──────────┬───────────────────┘
                  gRPC 长连 │ 心跳 + 指令
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
       Agent@工作机1   Agent@工作机2  ... Agent@工作机N
       deepsea-agent   deepsea-agent      deepsea-agent
```

产物分两个二进制:

- `deepsea-server`: 控制面, 部署在 3 台控制节点(单节点起步时先 1 台)
- `deepsea-agent`: Agent, 部署在所有被管理机器(含控制节点自身)

前端 `dist/` 由控制面节点的 nginx 托管, 反代 `/api` 到 8080。v0.4 起, 也可通过 SSH 自动注入功能在控制面 UI 上一键推送二进制 + systemd 配置到目标服务器, 自动拉起服务并加入集群, 无需手动 scp + systemctl。

---

## 二、端口规划

| 端口 | 服务 | 暴露范围 | 说明 |
|------|------|---------|------|
| 80 / 443 | nginx | 所有访问者 | 前端 + API 反代 |
| 8080 | deepsea-server HTTP | 本机 + 控制面节点间 | REST API, 经 nginx 反代后可不对外 |
| 9090 | deepsea-server gRPC | 所有 Agent | Agent 连接入口 |
| 7000 | Raft 节点间通信 | 控制面节点之间 | 选主/日志复制, 不对 Agent 开放 |

防火墙原则: 9090 对所有 Agent 开放, 7000 仅控制面节点互通, 8080 默认只本机(nginx 反代)。

## 三、构建产物

### 3.1 后端交叉编译(从 Windows 或任意开发机)

目标系统 Linux amd64。Go 交叉编译无需 CGO(本项目纯 Go 依赖)。

```powershell
# 在 server/ 目录
$env:GOROOT='F:\Go\go'
$env:GOPROXY='https://goproxy.cn,direct'
$env:GOCACHE='D:\work\projects\deep-sea-ops\.cache\go-build'
$env:GOOS='linux'
$env:GOARCH='amd64'
$env:CGO_ENABLED='0'

# 控制面
& 'F:\Go\go\bin\go.exe' build -buildvcs=false -ldflags="-s -w" -o deepsea-server ./cmd/server
# Agent
& 'F:\Go\go\bin\go.exe' build -buildvcs=false -ldflags="-s -w" -o deepsea-agent ./cmd/agent
```

`-ldflags="-s -w"` 去掉调试信息, 体积减半(约 19MB → 12MB)。`CGO_ENABLED=0` 产出纯静态二进制, 任意 Linux 发行版可直接跑, 无 glibc 版本依赖。

Linux/Mac 开发机同理, 去掉 GOROOT 全路径用 `go` 即可。

### 3.2 前端构建

```bash
cd web
npm install
npm run build      # 产物在 web/dist/
```

`dist/` 是纯静态文件, 拷到控制面节点由 nginx 托管。

### 3.3 产物清单

一次完整构建产出:

```
deploy/
├── deepsea-server          控制面二进制
├── deepsea-agent           Agent 二进制
├── dist/                   前端静态文件
├── config/
│   ├── server.yaml         控制面配置(见第六节)
│   └── agent.yaml          Agent 配置
├── systemd/
│   ├── deepsea-server.service
│   └── deepsea-agent.service
└── scripts/
    ├── bootstrap-agent.sh  Agent 一键安装
    └── join-control.sh     控制面加节点(v0.2)
```

建议在开发机构建好后, 整个 `deploy/` 目录打包 `tar -czf deepsea-ops-deploy.tar.gz deploy/` 分发。

## 四、单节点控制面首部署(v0.1)

以控制节点 `ctrl-01` (示例 IP 10.0.0.11) 为例。

### 4.1 准备目录与用户

```bash
# 在 ctrl-01 上
sudo useradd -r -m -d /opt/deepsea -s /usr/sbin/nologin deepsea
sudo mkdir -p /opt/deepsea/{bin,data,logs,config}
sudo chown -R deepsea:deepsea /opt/deepsea
```

目录用途:

- `bin/`: 二进制
- `data/`: Raft 日志/快照、bbolt 文件(持久化数据, 必须保留)
- `logs/`: 日志
- `config/`: 配置文件

### 4.2 上传二进制

```bash
# 从开发机
scp deploy/deepsea-server deploy/deepsea-agent ctrl-01:/tmp/
scp -r deploy/dist ctrl-01:/tmp/
scp deploy/config/server.yaml deploy/systemd/deepsea-server.service ctrl-01:/tmp/

# 在 ctrl-01
sudo mv /tmp/deepsea-server /tmp/deepsea-agent /opt/deepsea/bin/
sudo chmod +x /opt/deepsea/bin/deepsea-*
sudo mv /tmp/dist /opt/deepsea/dist
sudo mv /tmp/server.yaml /opt/deepsea/config/
```

### 4.3 控制面配置 /opt/deepsea/config/server.yaml

```yaml
node_id: ctrl-01
raft:
  data_dir: /opt/deepsea/data
  addr: 10.0.0.11:7000        # 本机内网 IP
http:
  addr: :8080
grpc:
  addr: :9090
log:
  level: info
  file: /opt/deepsea/logs/server.log
```

注: 当前 v0.1 代码硬编码了 `raft-data` 和 `127.0.0.1:7000`, 配置文件读取在 v0.2 落地。过渡期可直接用默认值, 把工作目录设为 `/opt/deepsea`。

### 4.4 systemd 服务 /etc/systemd/system/deepsea-server.service

```ini
[Unit]
Description=DeepSea Ops Control Plane
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=deepsea
Group=deepsea
WorkingDirectory=/opt/deepsea
ExecStart=/opt/deepsea/bin/deepsea-server
Restart=on-failure
RestartSec=5s
LimitNOFILE=65536
StandardOutput=append:/opt/deepsea/logs/server.log
StandardError=append:/opt/deepsea/logs/server.err.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo mv /tmp/deepsea-server.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now deepsea-server
sudo systemctl status deepsea-server
```

### 4.5 nginx 托管前端 + 反代 API

```bash
sudo apt install -y nginx    # Debian/Ubuntu; RHEL 用 dnf install nginx
sudo mv /opt/deepsea/dist/* /var/www/deepsea/   # 先 sudo mkdir -p /var/www/deepsea
```

`/etc/nginx/conf.d/deepsea.conf`:

```nginx
server {
    listen 80;
    server_name _;                     # 或控制面 VIP/域名

    root /var/www/deepsea;
    index index.html;

    # 前端单页应用: 所有非文件请求回退到 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反代到控制面 HTTP
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
```

访问 `http://ctrl-01/` 即可看到前端。`/api/` 自动转发到 8080。

### 4.6 防火墙

```bash
# ctrl-01
sudo ufw allow 80/tcp            # 前端访问
sudo ufw allow 9090/tcp          # Agent 连接(可限来源网段)
sudo ufw allow 7000/tcp          # Raft(限其他控制面节点)
# 8080 不对外, 仅 nginx 本机访问
```

## 五、Agent 部署

### 5.1 单台手动部署

以工作机 `worker-01` (10.0.0.21) 为例。

```bash
# 上传二进制
scp deploy/deepsea-agent worker-01:/tmp/

# 在 worker-01
sudo useradd -r -m -d /opt/deepsea -s /usr/sbin/nologin deepsea
sudo mkdir -p /opt/deepsea/{bin,logs,config}
sudo mv /tmp/deepsea-agent /opt/deepsea/bin/
sudo chmod +x /opt/deepsea/bin/deepsea-agent
```

Agent 配置 `/opt/deepsea/config/agent.yaml`:

```yaml
agent_id: worker-01
server: 10.0.0.11:9090        # 控制面 gRPC 地址(v0.2 多节点时可配多地址)
log:
  file: /opt/deepsea/logs/agent.log
```

systemd `/etc/systemd/system/deepsea-agent.service`:

```ini
[Unit]
Description=DeepSea Ops Agent
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=deepsea
Group=deepsea
WorkingDirectory=/opt/deepsea
ExecStart=/opt/deepsea/bin/deepsea-agent -id worker-01 -server 10.0.0.11:9090
Restart=on-failure
RestartSec=5s
StandardOutput=append:/opt/deepsea/logs/agent.log
StandardError=append:/opt/deepsea/logs/agent.err.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now deepsea-agent
sudo systemctl status deepsea-agent
```

启动后控制面 `/api/agents` 应出现该 Agent, 前端"Agent 节点"页可见。

### 5.2 批量部署(SSH 循环)

准备 `hosts.txt`:

```
worker-01 10.0.0.21
worker-02 10.0.0.22
worker-03 10.0.0.23
```

`scripts/bootstrap-agent.sh`:

```bash
#!/bin/bash
# 用法: ./bootstrap-agent.sh <agent.tar> <control_grpc_addr> <hosts.txt>
AGENT_PKG=$1
CTRL=$2
HOSTS=$3

while read -r name ip; do
  [ -z "$name" ] && continue
  echo "=== 部署 $name ($ip) ==="
  scp "$AGENT_PKG" "$ip:/tmp/deepsea-agent"
  ssh "$ip" bash -s <<EOF
    sudo useradd -r -m -d /opt/deepsea -s /usr/sbin/nologin deepsea 2>/dev/null
    sudo mkdir -p /opt/deepsea/{bin,logs}
    sudo mv /tmp/deepsea-agent /opt/deepsea/bin/
    sudo chmod +x /opt/deepsea/bin/deepsea-agent
    sudo tee /etc/systemd/system/deepsea-agent.service >/dev/null <<UNIT
[Unit]
Description=DeepSea Ops Agent
After=network-online.target
Wants=network-online.target
[Service]
Type=simple
User=deepsea
Group=deepsea
WorkingDirectory=/opt/deepsea
ExecStart=/opt/deepsea/bin/deepsea-agent -id $name -server $CTRL
Restart=on-failure
RestartSec=5s
StandardOutput=append:/opt/deepsea/logs/agent.log
StandardError=append:/opt/deepsea/logs/agent.err.log
[Install]
WantedBy=multi-user.target
UNIT
    sudo systemctl daemon-reload
    sudo systemctl enable --now deepsea-agent
EOF
done < "$HOSTS"
```

```bash
chmod +x scripts/bootstrap-agent.sh
./scripts/bootstrap-agent.sh deploy/deepsea-agent 10.0.0.11:9090 hosts.txt
```

前提: 部署机对 hosts.txt 中所有机器已配置 SSH 免密 sudo。后续 v0.4 会在控制面内置此流程, 导入名单即可自动部署。

## 六、扩展为 3 节点控制面集群(v0.2)

v0.1 单节点跑通后, 扩展到 3 节点获得容错。控制面节点 `ctrl-01/02/03` (10.0.0.11/12/13)。

### 6.1 部署 ctrl-02、ctrl-03

重复第四节流程, 注意:

- `server.yaml` 的 `raft.addr` 用各自 IP
- `node_id` 各不相同
- nginx 都托管同一份 `dist/`

### 6.2 加入 Raft 集群

v0.2 会实现动态加节点 API。过渡期做法: 在 ctrl-02/03 启动时不 bootstrap, 而是让 ctrl-01 调用 Raft `AddVoter` 把它们加入:

```bash
# 通过控制面管理接口( v0.2 提供 )
curl -XPOST http://ctrl-01:8080/api/cluster/join \
  -d '{"id":"ctrl-02","addr":"10.0.0.12:7000"}'
curl -XPOST http://ctrl-01:8080/api/cluster/join \
  -d '{"id":"ctrl-03","addr":"10.0.0.13:7000"}'
```

加入后 Raft 自动复制日志, ctrl-02/03 成为 Follower。3 节点成型后, 杀掉 ctrl-01, ctrl-02/03 秒级选出新 Leader, Agent 重连。

### 6.3 入口代理(任意节点访问)

v0.4 实现。每台控制面节点跑轻量代理, 查询当前 Leader 转发请求。过渡期用 nginx upstream + 健康检查:

```nginx
upstream deepsea_ctrl {
    server 10.0.0.11:8080;
    server 10.0.0.12:8080 backup;
    server 10.0.0.13:8080 backup;
}
location /api/ {
    proxy_pass http://deepsea_ctrl;
}
```

注: 写请求最终都要到 Leader, upstream backup 模式在 Leader 切换时有秒级中断, v0.4 入口代理解决此问题。

控制面 `/opt/deepsea/data/`:

### 7.2 备份

```bash
# 控制面数据备份(在 Leader 上做)

```

建议 crontab 每日备份, 保留 7 天。注意: 只备份 Leader, Follower 数据可从 Leader 重建。

## 九、升级流程

### 9.1 控制面滚动升级(3 节点)

```bash
# 逐台升级, 每台之间观察 Leader 稳定
for node in ctrl-03 ctrl-02 ctrl-01; do
  scp deploy/deepsea-server $node:/tmp/
  ssh $node sudo systemctl stop deepsea-server
  ssh $node sudo mv /tmp/deepsea-server /opt/deepsea/bin/
  ssh $node sudo chmod +x /opt/deepsea/bin/deepsea-server
  ssh $node sudo systemctl start deepsea-server
  sleep 10    # 等 Raft 重新稳定
done
```

先升 Follower(ctrl-03/02), 最后升当前 Leader(ctrl-01)。Leader 升级会触发选举, 期间几秒不可写。Raft 日志格式向前兼容时滚动升级安全; 不兼容时需停机升级(少见)。

### 9.2 Agent 升级

Agent 无状态, 可直接批量:

```bash
while read -r name ip; do
  scp deploy/deepsea-agent $ip:/tmp/
  ssh $ip "sudo systemctl stop deepsea-agent && sudo mv /tmp/deepsea-agent /opt/deepsea/bin/ && sudo chmod +x /opt/deepsea/bin/deepsea-agent && sudo systemctl start deepsea-agent"
done < hosts.txt
```

Agent 升级期间被管理的 Java 进程不受影响(只是控制面暂时管不了它)。

## 十一、常见问题

**Q: Agent 连不上控制面?**
检查 9090 端口防火墙、`-server` 地址、控制面 `deepsea-server` 是否运行。Agent 日志会显示连接错误。

**Q: Raft 节点选不出 Leader?**
3 节点必须能互相访问 7000 端口。检查防火墙、各节点 `raft.addr` 配置是否用对了内网 IP(不能用 127.0.0.1, 多节点间无法访问)。

**Q: 重启控制面数据丢了?**
检查 `/opt/deepsea/data/` 是否存在且 `WorkingDirectory` 正确。Raft 靠这个目录恢复, 删了等于清库。

**Q: 前端能打开但 API 404?**
nginx 的 `/api/` 反代未生效, 检查 `proxy_pass` 是否指向 8080、控制面是否在跑。

**Q: 单节点怎么过渡到 3 节点?**
先把 v0.1 单节点跑稳, 再按第六节加节点。加节点时新节点 `data/` 必须为空, 否则 Raft 状态冲突。

## 十二、SSH 自动注入部署 (v0.4)

v0.4 起, 控制面内置 SSH 自动注入功能, 取代第五节的手动 Agent 部署脚本。

### 12.1 前置条件

1. 控制面已启动 (单节点或 3 节点集群)
2. 在"SSH 凭据"页添加目标服务器的 SSH 连接信息 (密码或私钥, AES-GCM 加密存 Raft)
3. 设置环境变量 `MASTER_KEY` (SSH 凭据加密主密钥)
4. 本机有编译好的 `deepsea-server` / `deepsea-agent` 二进制

### 12.2 注入流程

1. 在"集群拓扑"页点击"注入新节点"
2. 选择角色: Raft 节点 (控制面) 或 Agent 节点 (工作机)
3. 选择 SSH 凭据, 填写节点 ID 和地址
4. 点击"开始注入", 控制面自动:
   - SSH 连接目标服务器
   - 上传二进制 + systemd 配置
   - 远程启动 systemd 服务
   - (Raft 节点) 调用 AddVoter 加入集群
   - (Agent 节点) 自动连 Leader gRPC

### 12.3 入口代理

v0.4 实现了入口代理, 任意控制面节点 IP 都能访问 UI:
- 读请求: 本地直接处理
- 写请求: 自动转发到当前 Leader
- Leader 切换时自动重连新 Leader

这意味着 nginx upstream + 健康检查不再是必需的, 浏览器直接打任意节点 IP:8080 即可。
